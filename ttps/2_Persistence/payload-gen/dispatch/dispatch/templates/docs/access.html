Overview
Built with offensive security in mind, Dispatch provides several methods of restricting access from unauthorized users:
  • Login Restrictions - Login restrictions are found under settings and can block all unauthorized source IP addresses from accessing the Dispatch web interface. This protection focuses on the privacy of the web interface and preventing login or viewing capabilities.
  • Access Restrictions - Access restrictions focus on the privacy of stored files and have the ability to deny requests based on source IP, User-Agent, or URL Parameters. This protection sits in-front of file permissions and requires all requests meet the criteria outlined in the Access Restrictions configurations.
Why? - this split configuration allows operators to whitelist target addresses while still protecting the admin interface.

Parameter Key Rotation?
This is the newest form of access control, which uses rotating parameters as key values to permit access. Even public files can be protected with key rotation for an easy, effective means of stopping blue teams from robbing your payloads!
